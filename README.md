1) Would you use a unit test to test the “message” feature of a messaging application? Why or why not? For this question, assume the “message” feature allows a user to write and send a message to another user.
    - I would not only use unit tests to test the message feature especially because sending a message has a lot of things working together like typing the message itseld, hitting send, if needed maybe saving it, sending it to someone else, and showing it on the screen. I think that a unit test is better for something smaller and in this case the best option would probably be integration or end to end testing 
2) Would you use a unit test to test the “max message length” feature of a messaging application? Why or why not? For this question, assume the “max message length” feature prevents the user from typing more than 80 characters.
    - Yes I would use a unit test for the max message length feature because it is one small rule to test. You can test that messages under 80 characters are allowed and messages over 80 characters are blocked
